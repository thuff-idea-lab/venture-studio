# Pre-Build Questions for Cocktail Menu Builder for Bar Managers

Answer these questions before handing the spec to your coding agent. For each question, a suggested default is provided — accept it or override it.

## Branding & Identity
### What will be the name of the product?
**Why it matters:** The name is essential for branding and marketing efforts.  
**Suggested default:** "Cocktail Menu Pro"

### What domain name will you use for the product?
**Why it matters:** A domain name is crucial for online presence and user accessibility.  
**Suggested default:** cocktailmenupro.com

### Do you have a logo design in mind, or should a simple text-based logo be created?
**Why it matters:** A logo helps establish brand identity and recognition.  
**Suggested default:** A simple text-based logo.

### What tagline will you use to describe the product?
**Why it matters:** A tagline succinctly communicates the product's value proposition.  
**Suggested default:** "Craft the Perfect Cocktail Menu with Ease"

## Product Behavior
### What core features must be included in the initial build?
**Why it matters:** Defining core features ensures the product meets user needs and expectations.  
**Suggested default:** Menu creation, ingredient-based suggestions, and presentation tips.

### How should the user interface handle ingredient availability (e.g., show unavailable ingredients)?
**Why it matters:** This affects user experience and how effectively users can create menus.  
**Suggested default:** Grey out unavailable ingredients and provide a tooltip with alternatives.

### Should users be able to save multiple versions of their cocktail menus?
**Why it matters:** This feature enhances usability and allows for experimentation.  
**Suggested default:** Yes, allow saving multiple versions.

## Monetization Setup
### What will be the price for the customizable cocktail menu template?
**Why it matters:** Pricing impacts sales strategy and revenue generation.  
**Suggested default:** $29.99 for the template.

### Will there be a free trial period, and if so, how long?
**Why it matters:** A free trial can attract initial users and encourage purchases.  
**Suggested default:** 14 days free trial.

## Content & Data
### What initial cocktail recipes and presentation tips will be included?
**Why it matters:** Initial content sets the stage for user engagement and satisfaction.  
**Suggested default:** 10 classic cocktail recipes with basic presentation tips.

### How will you source additional cocktail recipes and tips for future updates?
**Why it matters:** Ongoing content updates are crucial for keeping the product relevant and engaging.  
**Suggested default:** Curate from reputable cocktail recipe websites and user submissions.

## Accounts & Services
### Which payment processor will you use for transactions?
**Why it matters:** The payment processor affects transaction fees and user experience.  
**Suggested default:** Stripe.

### What analytics service will you use to track user engagement and sales?
**Why it matters:** Analytics are essential for understanding user behavior and improving the product.  
**Suggested default:** Google Analytics.

### Which email service provider will you use for user communications and marketing?
**Why it matters:** An email service is necessary for user engagement and support.  
**Suggested default:** Mailchimp.
