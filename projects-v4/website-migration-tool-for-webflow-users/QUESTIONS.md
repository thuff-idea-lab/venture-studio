# Pre-Build Questions for Website Migration Tool for Webflow Users

Answer these questions before handing the spec to your coding agent. For each question, a suggested default is provided — accept it or override it.

## Branding & Identity

### What will be the name of the product?
**Why it matters:** The name is crucial for branding and marketing efforts, influencing how users perceive the tool.  
**Suggested default:** "Webflow Migration Assistant"

### What domain will you use for the product?
**Why it matters:** A clear and memorable domain is essential for user access and marketing.  
**Suggested default:** "webflowmigrationassistant.com"

### Do you have a logo design in mind, or should we use a placeholder?
**Why it matters:** A logo contributes to brand identity and user trust; a placeholder can be used temporarily.  
**Suggested default:** Use a simple placeholder logo until a final design is created.

### What tagline will you use to describe the product?
**Why it matters:** A tagline succinctly communicates the product's value proposition to potential users.  
**Suggested default:** "Streamline Your Webflow Migrations Effortlessly"

## Product Behavior

### What file formats should the tool accept for uploads?
**Why it matters:** Defining accepted formats affects the user experience and the technical implementation of the upload feature.  
**Suggested default:** Accept CSV and JSON formats.

### How should the output be structured after data cleanup?
**Why it matters:** The structure of the output is critical for ensuring compatibility with Webflow and user expectations.  
**Suggested default:** Output should be a structured JSON format compatible with Webflow's import requirements.

### What error handling should be implemented for invalid uploads?
**Why it matters:** Clear error handling improves user experience and helps users correct their mistakes.  
**Suggested default:** Display a user-friendly error message indicating the issue with the upload.

## Monetization Setup

### Which payment processor will you use for transactions?
**Why it matters:** The choice of payment processor affects transaction fees, user experience, and integration complexity.  
**Suggested default:** Use Stripe for payment processing.

### Will you offer a free trial, and if so, for how long?
**Why it matters:** A free trial can attract initial users and encourage them to convert to paying customers.  
**Suggested default:** Offer a 14-day free trial.

### What will be the refund policy for users?
**Why it matters:** A clear refund policy builds trust and can influence purchasing decisions.  
**Suggested default:** No refunds after the migration project is processed.

## Content & Data

### What initial content will be provided on the website?
**Why it matters:** Initial content is essential for user onboarding and marketing; it sets the tone for the product.  
**Suggested default:** Include an overview of the tool, a how-to guide, and FAQs.

### Do you have specific copy for the landing page, or should we create it?
**Why it matters:** Well-crafted copy is crucial for conversion rates and user understanding.  
**Suggested default:** Create initial copy based on the product's features and benefits.

## Accounts & Services

### Which analytics service will you use to track user behavior?
**Why it matters:** Analytics are vital for understanding user engagement and improving the product.  
**Suggested default:** Use Google Analytics for tracking.

### What email service provider will you use for user communications?
**Why it matters:** The choice of email service affects deliverability, user engagement, and marketing capabilities.  
**Suggested default:** Use Mailchimp for email communications.

### Will you need a customer support tool, and if so, which one?
**Why it matters:** A support tool is essential for managing user inquiries and improving customer satisfaction.  
**Suggested default:** Use Zendesk for customer support.
