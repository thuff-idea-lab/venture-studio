# Pre-Build Questions for Minimalist Task Management Dashboard for ADHD Users

Answer these questions before handing the spec to your coding agent. For each question, a suggested default is provided — accept it or override it.

## Branding & Identity

### What will be the name of the template?
**Why it matters:** The name is crucial for branding and marketing, influencing how users perceive the product.  
**Suggested default:** "ADHD Task Manager"

### Do you want a custom domain for the product?
**Why it matters:** A custom domain can enhance credibility and make it easier for users to find and remember your product.  
**Suggested default:** Yes, use a domain like "adhdtaskmanager.com".

### Will you create a logo for the template?
**Why it matters:** A logo helps establish brand identity and can make the product more visually appealing.  
**Suggested default:** Yes, use a simple text-based logo with the product name.

### What tagline will you use for the product?
**Why it matters:** A tagline succinctly communicates the product's value proposition and can attract potential users.  
**Suggested default:** "Simplify Your Tasks, Focus Your Mind."

## Product Behavior

### What essential features must be included in the template?
**Why it matters:** Defining core features ensures the product meets user needs and avoids unnecessary complexity.  
**Suggested default:** Task list, priority tagging, and due dates.

### How should the user interface handle completed tasks?
**Why it matters:** Users may prefer different ways to manage completed tasks, impacting their experience with the product.  
**Suggested default:** Hide completed tasks but allow users to view them if desired.

### Should there be a mobile-friendly version of the template?
**Why it matters:** Many users may access the template on mobile devices, and a mobile-friendly design can enhance usability.  
**Suggested default:** Yes, ensure the template is responsive for mobile use.

## Monetization Setup

### What will be the price of the template?
**Why it matters:** Pricing affects sales and perceived value, and it should align with user expectations and market standards.  
**Suggested default:** $19.99 for the template.

### Will you offer a money-back guarantee?
**Why it matters:** A guarantee can increase user trust and reduce purchase hesitation, potentially boosting sales.  
**Suggested default:** Yes, a 30-day money-back guarantee.

## Content & Data

### What initial content or instructions will be included in the template?
**Why it matters:** Clear instructions help users understand how to use the template effectively, improving their experience.  
**Suggested default:** A brief user guide and tips for managing tasks effectively.

### Do you want to include any sample tasks or data in the template?
**Why it matters:** Providing sample data can help users visualize how to use the template and get started quickly.  
**Suggested default:** Yes, include 5 sample tasks.

## Accounts & Services

### Will you set up a payment processing account (e.g., Stripe)?
**Why it matters:** A payment processing account is essential for handling transactions and ensuring smooth purchases.  
**Suggested default:** Yes, set up a Stripe account.

### Do you need an email provider for customer communications?
**Why it matters:** An email provider is necessary for sending purchase confirmations, updates, and support communications.  
**Suggested default:** Yes, use Mailchimp for email communications.

### Will you implement analytics to track user engagement?
**Why it matters:** Analytics provide insights into user behavior, helping you improve the product over time.  
**Suggested default:** Yes, set up Google Analytics for tracking.
