# Pre-Build Questions for Unified Email Management Tool for Outlook and Gmail

Answer these questions before handing the spec to your coding agent. For each question, a suggested default is provided — accept it or override it.

## Branding & Identity
### What will be the name of the product?
**Why it matters:** The product name is crucial for branding and user recognition.  
**Suggested default:** "Unified Mail Manager"

### What domain will you use for the product's website?
**Why it matters:** A domain is essential for online presence and user access.  
**Suggested default:** "unifiedmailmanager.com"

### Do you have a logo design in mind, or should a placeholder be created?
**Why it matters:** A logo is key for brand identity and user trust.  
**Suggested default:** "Use a simple text-based logo for now."

### What tagline will you use to describe the product?
**Why it matters:** A tagline helps convey the product's value proposition quickly.  
**Suggested default:** "Manage all your emails in one place."

## Product Behavior
### What tagging system will users be able to use (e.g., color-coded, custom labels)?
**Why it matters:** The tagging system affects user experience and organization capabilities.  
**Suggested default:** "Allow users to create custom labels with no color restrictions."

### Will users be able to set up multiple custom signatures for different accounts?
**Why it matters:** Custom signatures enhance professionalism and personalization in emails.  
**Suggested default:** "Yes, allow multiple signatures per account."

### How will the app handle duplicate emails from both accounts?
**Why it matters:** Handling duplicates affects user experience and email management efficiency.  
**Suggested default:** "Show duplicates but allow users to merge or delete them."

## Monetization Setup
### Which payment processor will you use for subscriptions?
**Why it matters:** Choosing a payment processor is essential for managing subscriptions and revenue.  
**Suggested default:** "Stripe"

### Will there be a free trial period for new users?
**Why it matters:** A free trial can encourage user adoption and reduce barriers to entry.  
**Suggested default:** "Yes, offer a 14-day free trial."

### What features will be included in the free vs paid versions?
**Why it matters:** Defining feature boundaries helps clarify the value of the paid subscription.  
**Suggested default:** "Free version includes basic email management; paid version includes custom signatures and advanced tagging."

## Content & Data
### What initial content will be included in the app (e.g., help articles, FAQs)?
**Why it matters:** Initial content helps users understand and utilize the app effectively.  
**Suggested default:** "Include basic FAQs and a getting started guide."

### What tone and style should the copy use throughout the app?
**Why it matters:** Consistent tone and style enhance user experience and brand identity.  
**Suggested default:** "Friendly and professional."

## Accounts & Services
### Which analytics service will you use to track user behavior?
**Why it matters:** Analytics are crucial for understanding user engagement and improving the product.  
**Suggested default:** "Google Analytics"

### Will you need to set up a mailing service for user communications (e.g., newsletters, updates)?
**Why it matters:** A mailing service is important for user engagement and retention.  
**Suggested default:** "Yes, use Mailchimp for email communications."
