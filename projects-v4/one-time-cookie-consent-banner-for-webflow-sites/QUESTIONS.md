# Pre-Build Questions for One-Time Cookie Consent Banner for Webflow Sites

Answer these questions before handing the spec to your coding agent. For each question, a suggested default is provided — accept it or override it.

## Branding & Identity

### What will be the name of the cookie consent banner template?
**Why it matters:** The name will be used in marketing materials and within the Webflow community, influencing first impressions.  
**Suggested default:** "Webflow Cookie Consent Banner"

### Do you want a custom domain for the product landing page? If so, what will it be?
**Why it matters:** A dedicated domain can enhance branding and make it easier for users to find the product.  
**Suggested default:** "webflowcookieconsent.com"

### Will you create a logo for the product? If yes, what style or elements do you envision?
**Why it matters:** A logo can help establish brand identity and make the product more recognizable.  
**Suggested default:** Simple text-based logo with a cookie icon.

### What tagline do you want to use for the product?
**Why it matters:** A tagline succinctly communicates the product's value proposition and can be used in marketing.  
**Suggested default:** "Your one-time solution for cookie compliance."

## Product Behavior

### What customization options should users have for the banner (e.g., colors, text, position)?
**Why it matters:** Customization options can enhance user satisfaction and make the product more appealing to a broader audience.  
**Suggested default:** Users can customize colors and text.

### Should the banner be dismissible, and if so, how should it behave after dismissal (e.g., remember the choice, show again after a time period)?
**Why it matters:** User experience and compliance with cookie consent regulations depend on how the banner behaves after interaction.  
**Suggested default:** The banner is dismissible and remembers the user's choice.

### What should happen if a user does not accept cookies? Should the banner disappear or remain visible?
**Why it matters:** This decision affects user experience and compliance with legal requirements regarding cookie consent.  
**Suggested default:** The banner remains visible until the user accepts cookies.

## Monetization Setup

### Which payment processor will you use for transactions (e.g., Stripe, PayPal)?
**Why it matters:** The choice of payment processor affects transaction fees, user experience, and integration complexity.  
**Suggested default:** Stripe

### Will you offer a money-back guarantee? If so, for how long?
**Why it matters:** A money-back guarantee can increase user trust and potentially boost sales.  
**Suggested default:** 14 days
